3 approches pour l'outil de pilotage des contrats

Sélectionne l'approche qui te convient — ou dis-moi ce que tu veux combiner

⚡
Skill Claude Code

A — Script IA interne

Un skill Claude Code piloté par l'équipe Monga. Import Excel → enrichissement fournisseur par IA → génération PDF lettres → export batch. Pas d'interface client.

Pour

  • Très rapide à déployer
  • 0 infra à gérer
  • IA-native

Contre

  • Usage interne uniquement
  • Pas d'accès client
  • Pas de tableau de bord

Adapté si : usage Monga only, MVP très rapide

🌐
App Web Classique

B — App web full-stack

Next.js + PostgreSQL + API REST. Multi-tenant : Monga admin + accès clients. Upload Excel, dashboard, alertes, génération lettres, multi-CTA envoi.

Pour

  • Interface client complète
  • Standard, maintenable
  • Multi-tenant natif

Contre

  • Plus long à construire
  • Infra + BDD à gérer
  • Enrichissement manuel

Adapté si : produit long terme, équipe dev dédiée

RECOMMANDÉ
🚀
App Web + Claude API

C — App web IA-augmentée

Next.js + Supabase (BDD + Auth + Storage) + Claude API. Interface moderne multi-tenant avec IA embarquée pour enrichissement fournisseurs et génération intelligente des lettres.

Pour

  • Accès Monga + clients
  • Enrichissement auto IA
  • Lettres personnalisées par IA
  • Infra rapide via Supabase

Contre

  • Coût API Claude
  • Intégrations tierces (AR24, Maileva)

Meilleur équilibre vitesse × puissance × expérience client

Approche sprint recommandée : Sprint 1 = import + dashboard + alertes échéances. Sprint 2 = enrichissement fournisseur + génération lettres. Sprint 3 = envoi multi-CTA (PDF / LRE / postal).
